> 最終更新: 2026-03-18

## 1) 前提（固定）

| 項目               | 内容                                                         |
| ------------------ | ------------------------------------------------------------ |
| ターゲット         | 就活開始〜選考中の大学生（応募社数が増えて締切管理が破綻し始めた層） |
| 1行ベネフィット    | 締切を登録するだけで、出し忘れゼロで就活を進められる          |
| 主チャネル         | X（旧Twitter） — 拡散性・就活タグ文化                         |
| 副チャネル         | TikTok / Instagram Reels — 30秒デモ動画                       |

| 週次優先KPI        | Signup数 / Activation率（24h以内に2件登録）                  |

## 3) 改善実験（今週3本）

> 1実験 = 1変数のみ変更。並行実施しない（原因が特定できなくなる）。
> 計測イベント定義は [docs/ANALYTICS_SPEC.md](ANALYTICS_SPEC.md) を参照。

---

### 実験A：LP CTAボタン文言の変更

| 項目           | 内容                                                                                     |
| -------------- | ---------------------------------------------------------------------------------------- |
| **仮説**       | 「無料で始める」より「30秒で締切を登録する」の方が具体的な行動が想像でき、クリック率が上がる |
| **変更内容**   | LPのメインCTAボタン文言を `無料で始める` → `30秒で締切を登録する（無料）` に変更          |
| **実施期間**   | Day1〜Day4（4日間）                                                                       |
| **計測イベント** | `lp_cta_click`（Analytics Spec §3） / signup_users                                     |
| **成功条件**   | LP→signup転換率が変更前比 **+20%以上**（絶対値で CTR 15% → 18%以上）                    |
| **失敗時の学び** | 文言より「LPのファーストビュー画像/コピー」が律速している可能性を検証する               |

---

### 実験B：サインアップ直後のオンボーディングプロンプト

| 項目           | 内容                                                                                        |
| -------------- | ------------------------------------------------------------------------------------------- |
| **仮説**       | サインアップ後にダッシュボードへ飛ばすだけだと「何をすれば良いかわからない」で離脱している。「まず1件登録してみて」のバナーを出すと Activation率が上がる |
| **変更内容**   | ダッシュボード初回訪問時（アイテム0件）に「最初の締切を登録してみましょう 👉」バナー（`/deadline/new` へのリンク）を表示 |
| **実施期間**   | Day1〜Day7（7日間）                                                                          |
| **計測イベント** | `deadline_created`（first item） / signup→activation funnel                               |
| **成功条件**   | Activation率（24h以内に2件登録）が **35%以上**（PRD目標値に到達）                            |
| **失敗時の学び** | バナーのクリック率を見て「認知はしているが登録まで至らない」なら入力フォームの簡素化を検討 |

---

### 実験C：通知メール件名の緊迫感強化

| 項目           | 内容                                                                                    |
| -------------- | --------------------------------------------------------------------------------------- |
| **仮説**       | 通知メールの件名が「締切のお知らせ」という汎用表現だと開封されにくい。企業名＋残り時間を入れると開封率が上がり、ダッシュボード訪問→ステータス更新につながる |
| **変更内容**   | 件名を `【締切のお知らせ】` → `【あと24時間】{企業名} の {種別} 締切が迫っています` に変更（`src/lib/mailer/` 内のテンプレ） |
| **実施期間**   | Day2〜Day7（通知が発火するユーザーが対象）                                               |
| **計測イベント** | メール開封率（Resend dashboard） / `dashboard_viewed`（通知経由） / `deadline_status_updated` |
| **成功条件**   | 通知メール起点の `deadline_status_updated` 率が **+30%以上**（現状ベースラインと比較）   |
| **失敗時の学び** | 件名より「送信タイミング（72h/24h/3h）」の組み合わせが重要かを次週実験する             |
