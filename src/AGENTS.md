# AGENTS.md — src/ エンジニアリングルール

このファイルは `src/` ディレクトリに適用される**局所ルール**です。
実装時のコーディング規約と品質基準を定義します。

## エンジニアリングルール

- 言語は **TypeScript を優先** する
- 可読性優先：関数は小さく、責務を分ける
- 公開 API / 入力がある箇所は **バリデーション** を入れる
- 秘密情報はハードコードしない（環境変数を使用し、`.env.example` に記載）

## ディレクトリ構成の方針

| ディレクトリ        | 役割                                     |
| :------------------ | :--------------------------------------- |
| `src/app/`          | Next.js App Router のルートとページ      |
| `src/features/`     | ドメインごとのビジネスロジック           |
| `src/lib/`          | 横断的なユーティリティ・外部クライアント |
| `src/config/`       | アプリ共通設定                           |
| `src/middleware.ts` | Edge ミドルウェア                        |

## 品質の最低基準（MVPでも守る）

- エラー時の挙動（空状態/権限/通信失敗）を最低限実装する
- クリティカル導線（登録→初回成功体験→課金/申込など）に限って、最小のテスト計画またはテストコードを用意する
- テストファイルは対象モジュールの `__tests__/` に配置する
- ログ/計測が必要なら `src/features/analytics/` を通じて計測する
